# P122 Evidence — Safe Remediation Runner

This folder captures representative executor outcomes from fixed allowlisted
actions in P122.

## Files

- `safe-execution.json` — successful non-destructive execution
  (`run_health_snapshot`)
- `approval-required-rejection.json` — deterministic rejection when approval is
  missing (`prepare_quarantine_request`)
- `blocked-rejection.json` — deterministic policy-blocked rejection
  (`run_log_ingest` in `lockdown` mode)
- `sample-audit.jsonl` — append-only audit events generated by the sample runs

## Redaction/Audit Notes

- Request payloads are passed through policy redaction patterns before evidence
  bundling and audit payload emission.
- Audit events include policy decision, approval state, execution status,
  rollback metadata, and evidence bundle reference.
- Raw secret-like values are not stored in evidence/audit payloads.
